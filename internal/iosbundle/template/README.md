# __MAR_APP_NAME__ — iOS

This Xcode project was generated by `mar build --target ios`. **Do
not edit the Swift sources by hand** — every file in `Sources/` is
overwritten on the next build. Customizing your app means changing
your mar code, not the iOS scaffolding.

## Open & build

```sh
open __MAR_APP_NAME__.xcodeproj
```

In Xcode: set your signing team under **Signing & Capabilities**,
pick a simulator or device, press ▶ Run.

That's the whole loop. `mar build --target ios` already produced
the `.xcodeproj` for you — no manual generation step.

### When you need to regenerate

After editing your mar code (or pulling a new `mar` release that
ships updated Swift scaffolding):

```sh
mar build --target ios          # regenerates Sources/ + .xcodeproj
```

Xcode picks up the changes the next time you build. No
re-installation needed.

## How it works

The Swift app is a generic shell. On cold start it:

1. Reads the embedded `Resources/program.json` snapshot (instant
   first paint).
2. Fetches `/_mar/program.json` from the backend (background
   refresh).
3. Runs the user's `main`, capturing the page list.
4. Mounts each `Page` natively via SwiftUI.

Whatever services and routes your mar app exposes show up as native
UI without any iOS code changes. That's the **OTA model** — change
your mar code, restart the iOS app, see new behavior. No App Store
re-submission.

## Connecting to the backend

| Build configuration | Backend used |
|---|---|
| **Debug** (▶ Run from Xcode) | Bonjour finds `_mar._tcp` on the LAN first; falls back to `__MAR_DEFAULT_BASE_URL__` |
| **Release** (Archive / TestFlight / App Store) | `__MAR_DEFAULT_BASE_URL__` only (Bonjour is compiled out) |

The release URL comes from `ios.serverUrl` in `mar.json`. Bundle
identity (`bundleId`, `displayName`, `marketingVersion`, `buildNumber`)
all live there too — change any value in `mar.json` and rebuild:

```sh
mar build --target ios
```

## Troubleshooting

**"Signing for X requires a development team."**
Xcode → project navigator → target → **Signing & Capabilities** →
pick your team. Required even for simulator builds since iOS 17.

**"No such module 'X'"** after pulling new mar code.
Run `mar build --target ios` again — the Sources changed but the
`.xcodeproj` reference list is stale until regenerated.

**App stuck on "Loading…" in Debug.**
Check that `mar dev` is running on a machine on the same Wi-Fi as
your device/simulator. On first launch, iOS prompts for Local
Network permission — if you tapped Deny, toggle it back on under
**Settings → Privacy → Local Network → __MAR_APP_NAME__**.

**Archive build connects to `localhost:3000`.**
You shipped without setting `ios.serverUrl` in `mar.json`. Set it,
then `mar build --target ios` and archive again.
